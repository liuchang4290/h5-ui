# 0001 Docs Stack V0

## Context

当前项目是内部 H5 React 组件库，`v0` 只有 `Button`。我们需要的不是通用组件开发工具，而是一套自己完全可控的说明站：

- 能写清楚组件定位、用法、状态和 API
- 能直接放真实 React 组件示例
- 能体现品牌和设计气质
- 不被第三方文档框架的默认壳子限制

## Tried

第三方文档框架已经试过，功能上没问题，但不满足当前诉求：

- 默认壳子工具感太强
- 想做出更强的品牌感，需要持续覆盖大量默认 UI
- 当前组件数量很少，没必要先引入一整套偏“组件开发平台”的体系

## Decision

`v0` 最终选择：

- `apps/playground` 继续负责自由联调
- `apps/docs` 改为自定义 `React + Vite` 文档站

这意味着：

- 不依赖第三方文档框架
- 页面结构、视觉风格、导航方式、示例排版都由仓库自己掌控
- 文档站直接使用 `@xbotgo/ui` 里的真实组件和 tokens

## Why

选择自定义说明站的原因：

- 更适合“酷炫但简单”的品牌化表达
- 对只有少量组件的早期阶段更轻
- 不需要为了改外观而和第三方默认布局做长期博弈
- 后续如果组件数量增加，仍然可以继续按同一套页面结构扩展

## Current Scope

`v0` 文档站至少包含：

- 首页品牌区
- 设计原则
- 设计色板
- `Button` 实时预览
- `Button` 使用示例
- `Button` API 表
- 路线图

## Follow-up

后续随着组件数量增长，再考虑：

- 是否需要组件列表页和详情页拆分
- 是否加入全文搜索
- 是否加入代码复制、在线切换主题、版本切换

当前阶段，保持轻量、直接、可控，比引入更重的文档生态更合适。
